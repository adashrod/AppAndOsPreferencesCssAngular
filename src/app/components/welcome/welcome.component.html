Handling user preferences in CSS 3 ways

This is a proof-of-concept for 3 different ways to reconcile operating system preferences with application preferences.

There are OS preferences that can queried in CSS and JS to show different user experiences; these include reduced motion, contrast, dark mode. and others. It's simple to use media queries to show different user experiences based on these preferences, or to provide a setting in your app to choose their preference, but you might want to let users utilize both, so that they can choose "always on", "always off", or "defer to OS". Here, we show 3 different ways to do this.

They vary in putting logic in CSS vs. JavaScript, and relying on newer CSS features that are not yet widely supported.

This uses the reduced motion feature as an example, but the other features would work the same way.
